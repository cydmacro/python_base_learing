### PPT 14: 第一个程序剖析
```
标题: Hello World 深度剖析

[代码展示]
print("Hello World")

[逐个解释]
print
  ├─ 是什么？函数（function）
  ├─ 作用？打印输出
  └─ 来自哪？Python内置函数

( )
  ├─ 是什么？括号
  ├─ 作用？函数的参数容器
  └─ 规则？必须是英文括号

"Hello World"
  ├─ 是什么？字符串（string）
  ├─ 作用？要打印的内容
  └─ 规则？必须用引号包裹

[底部]
💡 记住：函数名 + () + 参数
```

❌ print(Hello)      # 没有引号（Python以为Hello是变量）
✅ print("Hello")    # 正确

## 🎨 PPT设计建议

### 视觉风格
- **配色**: 蓝色系（专业）+ 橙色（活力）
- **字体**:
  - 标题：微软雅黑 Bold，32-36pt
  - 正文：微软雅黑，18-24pt
  - 代码：Consolas/Monaco，16-20pt
- **背景**: 浅灰或白色，避免太花哨

### 代码展示
- 使用代码高亮（Syntax Highlighting）
- 深色背景 + 浅色代码（类似IDE）
- 关键代码用黄色高亮
- 错误代码用红色标注

### 图标使用
- ✅❌⚠️💡🎯📚💻🚀 等emoji增加趣味性
- 但不要过度使用，每页3-5个即可

### 动画建议
- 代码逐行出现（配合讲解）
- 重点内容放大强调
- 避免过多花哨动画（影响注意力）

## 📝 演讲技巧

### 语速控制
- 正常语速：120-150字/分钟
- 重点内容：90-110字/分钟（放慢）
- 演示代码时：更慢，配合手指

### 互动节奏
- 每10分钟提问1次："大家跟上了吗？"
- 每完成一个知识点："谁来试试？"
- 鼓励举手提问

### 时间把控
- 严格按照PPT时间线
- 如果超时，省略次要PPT
- 保证核心内容讲透

### 应急预案
- 如果学员普遍听不懂：换个例子，放慢进度
- 如果学员觉得简单：增加挑战题
- 如果设备故障：提前准备备用电脑
